    2. 操作系统的定义
        1. 系统资源(软硬件资源)的管理者(基本功能)(可以查看任务管理器)
            ![[Pasted image 20260309190909.png]]
            1. 上例分别对应文件管理、存储器管理、处理器管理、设备管理等硬件资源
        2. 向上层提供方便易用的服务(封装)：对用户、对程序员
            1. 为用户：GUI、联机命令接口(命令行)：用户说一句，系统做一句；脱机命令接口(批处理命令)：用户说一堆，系统做一堆
            2. 为程序员/应用程序：提供程序接口(即系统调用)，程序接口本身由系统调用组成；一般用户只能通过用户程序间接使用
                ![[Pasted image 20260309191336.png]]
        3. 是最接近硬件的软件：需要实现对硬件机器的扩展
            ![[Pasted image 20260309191617.png]]
3. 操作系统的特征
    1. 并发：计算机系统中同时运行着多个程序(宏观上同时，微观上交替)(区别于并行)单核CPU只能并发，多核CPU才可以并行
    2. 共享：微观上也可能真的同时使用(打游戏时听音乐)
        ![[Pasted image 20260309192350.png]]
    3. 虚拟：用户感受到的内存大小、处理器个数、设备数量多于硬件实际
    4. 异步：由并发引起。由于各个程序交替执行，故单个程序的推进速度不可预知
    5. 并发和共享互为互为前提，而并发又是虚拟和异步的前提(没有并发就没有虚拟处理器、没有并发就不会出现异步)
        ![[Pasted image 20260309192425.png]]
4. 零碎知识点
    1. 程序执行之前要放到内存中才能执行
    2. 重入代码是指在并发环境下，同一段代码可以被多个执行流同时调用，而不会产生数据冲突或错误结果
5. 习题总结
    1. 文件不属于硬件资源，但属于计算机资源
    2. 操作系统管理文件，但不管理文件的内容，而是管理文件的逻辑组织、访问控制等内容。源程序属于文件，但是语义上侧重于其内容，故操作系统不管理源程序
    3. 外存属于硬件资源
    4. 系统调用的目的是请求系统服务
