2. 中缀表达式转后缀表达式
    1. 若中缀表达式运算顺序不唯一，则可以得到的后缀表达式也不唯一
        ![[Pasted image 20260206202209.png]]
    2. 添加**左优先**原则，即可保证结果唯一，从而确保转换算法的**确定性**。左优先原则即：如果**左边的运算可以先进行，那必须先算左边**。比如$A+B+C$,必须先算$A+B$,从而**保证了**运算顺序的唯一性
        ![[Pasted image 20260206202717.png]]
3. 中缀表达式转前缀表达式

        ![[Pasted image 20260206204256.png]]
4. 中缀表达式计算算法：同时使用操作数栈和运算符栈，结合中缀转后缀算法和后缀计算算法，逐个字符处理即可

5. 后缀表达式：本身是一个**递归结构**：op1 op2 运算符，其中op1和op2都是递归的后缀表达式或数字。后缀表达式**从左往右扫描**，右边的操作数更靠近运算符，所以会先算，即后到先算，故符合栈**LIFO**的特性

# 五、树与二叉树
1. 区分度为*m*的树和*m*叉树
    ![[Pasted image 20260212173220.png]]
2. 遍历表达式树：先序遍历得到前缀表达式、后序遍历得到后缀表达式、中序遍历再加括号得到中缀表达式
    ![[Pasted image 20260215152711.png]]
3. 线索二叉树：本质就是使用*pre*指针和*cur*指针，同时修改*visit*函数
    1. 先序线索化：按照根左右处理，由于处理时会修改根节点的左指针，可能导致根节点的左指针指向先序前驱，如果此时根节点是先序前驱的左孩子，就会导致死循环。因此需要加一个判别条件，只处理真正的左子树
        ![[Pasted image 20260216141952.png]]
    2. 中序线索化和后序线索化不会卡死，但是不要忘记处理最后一个节点
        ![[Pasted image 20260216142406.png]]
    3. 根据线索二叉树寻找前驱和后继
        1. 中序寻找前驱和后继都很方便
        2. 先序线索二叉树找后继容易，找前驱困难(需要使用三叉链表，并分类讨论)
            ![[Pasted image 20260216145712.png]]
            ![[Pasted image 20260216150146.png]]
        3. 后序线索二叉树找前驱容易，找后继困难(需要使用三叉链表，并分类讨论)
            ![[Pasted image 20260216150112.png]]
            ![[Pasted image 20260216150131.png]]    
4. 树转二叉树：使用孩子兄弟法；森林转二叉树：把树根看作兄弟
    ![[Pasted image 20260221104355.png]]
    ![[Pasted image 20260221104709.png]]
5. 树的遍历
    1. 先根：先访问根，再依次访问各子树，子树访问方式递归
    2. 后根：先依次访问子树，后根，子树递归
    3. 层次：同二叉树，用队列
    4. 也可以转为二叉树之后再遍历
6. 森林的遍历
    1. 先序：从左至右对每个子树进行先根遍历
    2. 中序：从左至右对每个子树进行后根遍历
    3. 也可以利用孩子兄弟法转二叉树再遍历
7. 并查集：将$n$个独立元素合并为一个集合，每个元素查找所属集合*O(n)*，合并操作*O(1)*，故总O(n$^2$)
